Notes

We tried to keep tables as simple as possible for the moment.

For context, on the NHS website:

  • tables never exceeded three columns, we didn't design for or test any tables over three columns.
  • The majority of NHS website tables are text, a few contain data (numerical).
  • We haven't uncovered any needs around re-ordering data in tables.

User needs

As a user, I need to easily find the information that relates to my situation, so that i can get better/ be informed.

What's currently being used

Our hypothesis

We believe

  • tables are hard to scan on the grey background, so we've put them in a white box
  • To aid scanning information we've added a hover colour on desktop screens

Accessibility

Following webaim.org's guidance for tables (https://webaim.org/techniques/tables/data) we suggest

  • Tables should have captions
  • Use the scope attribute to associate the data cells with the appropriate headers
  • Let the browser window determine the width of the table whenever possible, to reduce the horizontal scrolling required of those with low vision

Usage

Going forward

Tables will need captions. Currently on NHS.uk's wagtail we don't have the option to include captions so this will need to be explored further.

We'd also like to look at tables with more than three columns, tables that are mostly data and tables that could be reordered by a user.

What is success

Success would be a user being able to find the information they require.

How we measure success

Success can be measured by page depth, exit rate and page depth and also monitoring a feedback survey to find if any users mentioned not being able to find information that is presented in the table.
